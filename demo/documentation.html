Dokumentasi PDF Signature SDK

pdf-signature-jtp-sdk adalah pustaka rendering PDF dan penempatan tanda tangan elektronik (dan E-Materai) yang performan dan framework-agnostic. SDK ini dirancang agar mudah digunakan secara langsung di HTML murni menggunakan Vanilla JS, maupun menggunakan framework modern seperti Vue 3, React, dan Next.js.

Info Arsitektur: Pustaka ini membagi rendering halaman menjadi dua layer: layer dasar statis digambar dengan PDF.js ke Canvas, dan layer atas (overlay) interaktif dikelola langsung dengan HTML5 Canvas API murni untuk interaksi tanda tangan yang responsif tanpa library berat lainnya.

Fitur Utama

Rendering PDF.js Cepat

Menggunakan library standar industri PDF.js untuk kualitas render visual dokumen yang tajam dengan lazy-loading berbasis IntersectionObserver.

Tanda Tangan Interaktif

Pengguna dapat meletakkan, menyeret (drag & drop), serta mengubah ukuran (resize) placeholder tanda tangan secara intuitif di atas dokumen.

E-Materai Support

Dukungan bawaan untuk pembubuhan E-Materai elektronik dengan aspek rasio tetap (square) secara visual sesuai regulasi.

Kustomisasi UI Penuh

Tampilkan atau sembunyikan topbar, tombol, sidebar thumbnail, kontrol zoom, atau ganti label tombol teks lewat satu objek config.

Instalasi

Instal SDK ke dalam project Anda menggunakan package manager:

npm install pdf-signature-jtp-sdk

Panduan Integrasi Framework

Untuk penggunaan dasar menggunakan file HTML murni atau JS script standard, import fungsi pembuat viewer dan panggil dengan container CSS Selector.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Demo PDF Signature SDK</title>
  <!-- Import CSS default SDK -->
  <link rel="stylesheet" href="node_modules/pdf-signature-jtp-sdk/dist/style.css">
</head>
<body class="bg-slate-900">
  
  <!-- Wadah tempat viewer akan dirender -->
  <div id="my-pdf-container" style="width: 100%; height: 750px;"></div>

  <script type="module">
    import { createViewer } from './node_modules/pdf-signature-jtp-sdk/dist/pdf-signature-sdk.js';

    // Inisialisasi Viewer
    const viewer = createViewer({
      container: '#my-pdf-container',
      file: 'https://example.com/dokumen.pdf', // URL, File, Blob, atau ArrayBuffer
      scale: 1.5,
      theme: 'dark',
      signatureOptions: [
        { id: 'dir', label: 'Direktur Utama', image: 'https://example.com/sig-dir.png' },
        { id: 'emp', label: 'Karyawan', image: 'https://example.com/sig-emp.png' }
      ],
      estampOptions: [
        { id: 'estamp1', label: 'E-Materai Resmi', image: 'https://example.com/estamp.png' }
      ]
    });

    // Mendengarkan Event penempatan tanda tangan
    viewer.on('signaturePlaced', (sig) => {
      console.log('Tanda tangan ditempatkan pada koordinat:', sig.x, sig.y, 'halaman', sig.page);
    });

    // Mendengarkan Event perubahan halaman
    viewer.on('pageChanged', ({ page, total }) => {
      console.log(`Berpindah ke halaman ${page} dari total ${total}`);
    });
  </script>
</body>
</html>

Pustaka menyediakan adapter Vue 3 bawaan yang dapat langsung diimpor dari pdf-signature-jtp-sdk/vue.

<script setup>
import { ref } from 'vue';
import { PdfViewer } from 'pdf-signature-jtp-sdk/vue';
import 'pdf-signature-jtp-sdk/style.css';

const pdfFile = ref('https://mozilla.github.io/pdf.js/web/compressed.tracemonkey-pldi-09.pdf');
const scale = ref(1.5);
const theme = ref('dark');

const signatures = [
  { id: 'ceo', label: 'CEO Approval', image: 'https://upload.wikimedia.org/wikipedia/commons/f/f6/Signature_of_John_Hancock.svg' }
];

const viewerRef = ref(null);

function onSigPlaced(sig) {
  console.log('Tanda tangan ditambahkan:', sig);
}

function handleGetSigs() {
  if (viewerRef.value) {
    const allPlaced = viewerRef.value.getSignatures();
    console.log('Semua tanda tangan yang ditempatkan:', allPlaced);
  }
}
</script>

<template>
  <div class="p-6 bg-zinc-950 min-h-screen text-white">
    <div class="mb-4 flex gap-3">
      <button @click="handleGetSigs" class="px-4 py-2 bg-indigo-600 hover:bg-indigo-700 rounded-lg text-sm">
        Ambil Hasil Koordinat
      </button>
      <button @click="() => viewerRef?.clearSignatures()" class="px-4 py-2 bg-red-600 hover:bg-red-700 rounded-lg text-sm">
        Hapus Semua
      </button>
    </div>

    <PdfViewer
      ref="viewerRef"
      :file="pdfFile"
      :scale="scale"
      :theme="theme"
      :signature-options="signatures"
      :container-style="{ width: '100%', height: '700px' }"
      @signature-placed="onSigPlaced"
    />
  </div>
</template>

Pustaka menyediakan component dan hook React bawaan yang dapat langsung diimpor dari pdf-signature-jtp-sdk/react.

Peringatan Next.js SSR: Karena SDK ini mengakses DOM, Canvas API, dan variabel global browser (window), Anda harus me-render komponen ini secara Client-Side saja di Next.js dengan mematikan SSR.

1. Buat Komponen Wrapper Client (misal: Components/MyPdfViewer.jsx)

"use client";

import { useRef } from 'react';
import { PdfViewer } from 'pdf-signature-jtp-sdk/react';
import 'pdf-signature-jtp-sdk/style.css';

export default function MyPdfViewer() {
  const viewerRef = useRef(null);

  const signatureOptions = [
    { id: 'dir-1', label: 'Direktur Utama', image: 'https://example.com/sig.png' }
  ];

  const handleLogSigs = () => {
    if (viewerRef.current) {
      console.log('Signatures:', viewerRef.current.getSignatures());
    }
  };

  return (
    <div>
      <div className="flex gap-2 mb-4">
        <button onClick={handleLogSigs} className="bg-indigo-600 px-4 py-2 rounded text-sm text-white">Log Signatures</button>
        <button onClick={() => viewerRef.current?.clearSignatures()} className="bg-zinc-700 px-4 py-2 rounded text-sm text-white">Clear All</button>
      </div>

      <PdfViewer
        viewerRef={viewerRef}
        file="https://example.com/sample.pdf"
        scale={1.5}
        theme="dark"
        signatureOptions={signatureOptions}
        style={{ width: '100%', height: '700px' }}
        onSignaturePlaced={(sig) => console.log('Placed:', sig)}
        onPageChanged={({ page, total }) => console.log(`Halaman: ${page}/${total}`)}
      />
    </div>
  );
}

2. Panggil Dengan Dynamic Import Di Halaman (misal: App/Page.jsx)

"use client";

import dynamic from 'next/dynamic';

// Nonaktifkan Server-Side Rendering (SSR) untuk Komponen PDF Viewer

      <h1 className="text-2xl font-bold mb-6">Dokumen Approval</h1>
      <div className="border border-zinc-800 rounded-xl p-4 bg-zinc-900/40">
        <PDFViewerDynamic />
      </div>
    </main>
  );
}

Skema Konfigurasi SDK

Opsi konfigurasi lengkap saat memanggil fungsi createViewer(config) atau pada component props.

Properti Tipe Data Default Keterangan
container string | HTMLElement null Selector CSS (misal: '#app') atau Node HTML langsung sebagai wadah render.
file File | Blob | string | ArrayBuffer null Sumber dokumen PDF yang akan langsung dimuat secara otomatis.
scale number 1.5 Rasio pembesaran render awal (Zoom). Rentang yang diizinkan: 0.25 hingga 4.0.
theme 'light' | 'dark' 'light' Mode tema visual bawaan viewer. Dapat dirubah secara dinamis.
disabled boolean false Mengunci / menonaktifkan seluruh tombol aksi dan interaksi overlay canvas (menambahkan class .psdk-disabled).
disableDragging boolean false Mengunci pergeseran (drag) dan perubahan ukuran (resize) seluruh tanda tangan dan E-Materai. Tombol silang tetap aktif.
disableDrag boolean false Mengunci pergeseran (drag) seluruh tanda tangan dan E-Materai, namun ukuran (resize) tetap bisa diubah. Tombol silang tetap aktif.
disableResize boolean false Mengunci ukuran (resize) seluruh tanda tangan dan E-Materai (menyembunyikan handle resize), namun pergeseran (drag) tetap bisa dilakukan. Tombol silang tetap aktif.
signatureOptions Array<object> [] Opsi tanda tangan yang muncul di modal, berisi: { id, label, image, group }.
estampOptions Array<object> [] Opsi E-Materai yang muncul di modal, berisi: { id, label, image, group }.
groupByCategory boolean false Mengaktifkan pengelompokan pilihan tanda tangan dan e-materai secara visual di dalam modal berdasarkan kategori kelompok (membaca property group pada setiap item opsi).
labels object {...} Kustomisasi teks label UI: uploadBtn, signatureBtn, estampBtn, dll.
ui.topbar.upload boolean true Tampilkan tombol unggah file di topbar.
ui.topbar.signature boolean true Tampilkan tombol 'Add Signature' di topbar.
ui.topbar.eStamp boolean true Tampilkan tombol 'E-Materai' di topbar.
ui.topbar.pagination boolean true Tampilkan tombol Halaman Sebelumnya/Berikutnya dan indikator halaman.
ui.topbar.paginationInput boolean true Izinkan mengetik langsung nomor halaman pada indikator pagination.
ui.topbar.zoom boolean true Tampilkan kontrol zoom in, zoom out, dan fit-to-screen di topbar.
ui.topbar.themeToggle boolean true Tampilkan tombol beralih tema siang/malam di topbar.
ui.topbar.customComponent string | HTMLElement null Elemen khusus (custom HTML string atau DOM Element) yang dimasukkan di sisi kanan topbar.
ui.topbar.logo string | HTMLElement null Logo khusus (SVG/HTML string, image URL, atau DOM Element) untuk menggantikan icon dokumen default.
ui.topbar.brandText string | boolean null Nama brand kustom. Atur ke false atau string kosong untuk menyembunyikannya.
ui.topbar.style object null Object JavaScript yang berisi properti CSS inline override untuk wadah topbar.
ui.sidebar.thumbnails boolean true Tampilkan panel sidebar sebelah kiri berisi daftar thumbnail halaman PDF.

Referensi API (Instance Methods)

Metode imperatif yang diekspos oleh objek instance viewer yang dikembalikan oleh createViewer(), atau melalui React Ref / Vue Ref.

loadDocument(source)

Async

Memuat berkas PDF baru secara dinamis. Metode ini otomatis me-reset halaman aktif, menghapus tanda tangan yang ada, dan memperbarui panel thumbnail sidebar.

Parameters: source (File | Blob | String URL | ArrayBuffer) — Objek/Path file PDF.

nextPage() / prevPage()

Berpindah ke satu halaman berikutnya atau halaman sebelumnya jika tersedia.

goToPage(n)

Berpindah ke halaman tertentu berdasarkan nomor indeks halaman (indeks dimulai dari 1).

Parameters: n (number) — Nomor halaman target.
Returns: boolean — Mengembalikan true jika perpindahan halaman berhasil, dan false jika gagal (misal: nomor halaman di luar batas).

setScale(scale)

Async

Mengubah rasio pembesaran (zoom) dokumen secara dinamis dan me-render ulang halaman dokumen saat ini.

Parameters: scale (number) — Rasio skala baru (misal: 1.5 = 150%). Rentang: 0.25 sampai 4.0.

fitToScreen()

Async

Menghitung ukuran lebar dan tinggi wadah kontainer, lalu mengubah skala dokumen secara otomatis agar pas dengan ukuran layar.

Returns: number — Mengembalikan rasio skala yang baru dihitung dan diterapkan.

openSignatureModal() / openEStampModal()

Membuka popup modal pemilihan peran tanda tangan atau modal pemilihan E-Materai secara terprogram.

placeSignature(opts)

Menempatkan objek tanda tangan pada halaman secara langsung melalui kode.

Parameters: opts (object) — Opsi kustomisasi:

  • x (number, opsional): Koordinat X piksel (default: tengah halaman).
  • y (number, opsional): Koordinat Y piksel (default: tengah halaman).
  • page (number, opsional): Nomor halaman target (default: halaman saat ini).
  • label (string, opsional): Teks yang akan ditampilkan pada tanda tangan.
  • image (string, opsional): URL atau Base64 gambar tanda tangan.
  • disableDrag (boolean, opsional): Kunci pergeseran tanda tangan ini saja.
  • disableResize (boolean, opsional): Kunci perubahan ukuran tanda tangan ini saja.
  • disableDragging / locked (boolean, opsional): Kunci pergeseran & perubahan ukuran tanda tangan ini saja.
Returns: string — ID unik dari tanda tangan yang baru saja ditempatkan.

placeEStamp(opts)

Menempatkan objek E-Materai secara programmatik di halaman.

Parameters: opts (object) — Opsi penempatan:

  • x, y, page (opsional): Sama seperti tanda tangan.
  • image (string, opsional): Gambar khusus materai.
  • disableDrag (boolean, opsional): Kunci pergeseran E-Materai ini saja.
  • disableResize (boolean, opsional): Kunci perubahan ukuran E-Materai ini saja.
  • disableDragging / locked (boolean, opsional): Kunci pergeseran & perubahan ukuran E-Materai ini saja.

removeSignature(id)

Menghapus satu objek tanda tangan atau E-Materai tertentu berdasarkan ID uniknya.

Parameters: id (string) — ID objek yang ingin dihapus.

clearSignatures()

Menghapus seluruh tanda tangan dan E-Materai yang telah ditempatkan di semua halaman dokumen.

getSignatures() / getSignaturesByPage(page)

Mengambil data koordinat dari objek yang ditempatkan pengguna untuk dikirim ke backend proses penandatanganan.

Returns: Array<object> — Array berisi objek: { id, type, x, y, page, label, width, height }.

setPaginationLocked(locked)

Mengunci perpindahan halaman (membuat tombol prev/next dan input halaman tidak aktif).

Parameters: locked (boolean) — Nilai true untuk mengunci, false untuk membuka.

on(event, fn) / off(event, fn)

Menambahkan atau menghapus listener untuk event internal SDK (Event Bus).

updateConfig(partialConfig)

Memperbarui konfigurasi UI atau status interaktivitas secara dinamis (seperti merubah tema, bahasa, atau menyembunyikan tombol).

getCanvas(page) / getOverlayCanvas(page)

Mengambil referensi element HTML Canvas utama PDF.js atau Canvas Overlay core SDK untuk operasi tingkat lanjut.

destroy()

Menghancurkan instance viewer, menghapus seluruh event listener, memutus IntersectionObserver, dan membersihkan elemen DOM kontainer agar tidak terjadi kebocoran memori.

Getter Properties (Status Reaktif)

Status reaktif yang dapat dibaca kapan saja langsung dari instance viewer:

Property Tipe Data Keterangan
currentPage number Menunjukkan indeks halaman aktif saat ini (1-indexed).
totalPages number Total jumlah seluruh halaman dokumen PDF yang sedang dimuat.
currentScale number Skala perbesaran (zoom) aktif dokumen saat ini (misal: 1.5).
isPaginationLocked boolean Menunjukkan apakah perpindahan halaman sedang dikunci atau tidak.

Sistem Event (Event Bus)

Event yang dipancarkan oleh SDK dan dapat didengarkan menggunakan metode viewer.on(eventName, callback).

Nama Event Payload (Parameter Callback) Deskripsi Event
documentLoaded { totalPages: number } Dipicu ketika dokumen PDF baru telah selesai diparsing dan siap ditampilkan.
pageChanged { page: number, total: number, source: string } Dipicu saat halaman aktif berubah. source bernilai 'scroll' jika digulir atau 'api' jika dari navigasi.
signaturePlaced { id: string, type: 'signature', x: number, y: number, page: number, label: string, width: number, height: number } Dipicu ketika tanda tangan berhasil ditaruh ke dalam dokumen (baik lewat modal atau programmatik).
eStampPlaced { id: string, type: 'estamp', x: number, y: number, page: number, label: string, width: number, height: number } Dipicu ketika E-Materai berhasil ditaruh ke dalam dokumen.
signatureMoved { id: string, type: string, x: number, y: number, page: number, label: string, width: number, height: number } Dipicu ketika objek tanda tangan / E-Materai selesai diseret (drag) atau dirubah ukurannya (resize) oleh pengguna.
signatureRemoved { id: string } Dipicu ketika tanda tangan / E-Materai dihapus secara spesifik dengan mengklik tanda silang merah.

Isolasi Gaya & Kustomisasi CSS

Semua style bawaan SDK dibungkus dalam namespace CSS class .psdk-* untuk menghindari bentrokan gaya dengan CSS global aplikasi Anda. Anda dapat meng-override warna dasar bertema menggunakan CSS Variable berikut pada root container Anda: 

/* Meng-override warna accent utama (indigo) dan background viewer */
.my-container-class {
  --psdk-accent: #10b981;       /* Emerald hijau sebagai warna aksen */
  --psdk-accent-hover: #059669;
  --psdk-bg-base: #18181b;      /* Zinc 900 background */
  --psdk-border-color: rgba(255, 255, 255, 0.1);
  --psdk-font-family: 'Outfit', sans-serif;
}
